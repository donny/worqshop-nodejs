Architecture


Notes

1. Google App Engine acts as the scalable web front-end. They know better than me on how to scale user-facing web sites.

2. Amazon EC2 instances act as the back-end servers that execute Node.js applications.

3. The VPS from Prgmr acts as the Git server and manages the application deployment to the EC2 instances.

4. The VPS (256MB) is a bottleneck. But I started the Worqshop development there and in the future I may need to upgrade it or move it to Amazon EC2. Alternatively, I could deploy Git on top of GAE (similar to GitFarm) or just use Github with webhooks. I think I may go with the latter.

5. All HTTP requests to Node.js applications go through GAE. I want to keep the EC2 instances invisible to users.

6. I spent about a week (after hours) to implement this. I didn't have the time to implement the web UI (it's very bare). Thus, interactions are done through API calls.
